Local vs Remote
Local:
*Structured Environment: This provides a clear and physical seperation between work and home life.
*Organic Collaboration: This allows for consistent face-to-face interactions.
*Company Culture Immersion: This enables employees to experience the office atmosphere daily.

Remote:
*Flexibility and Autonomy: Helps offer greater control over work schedules and environments.
*No commute: Helps eliminate commuting time and expenses.
*Increased Productivity: Helps employers be hired from anywhere, which increases diversity and accesses specialized skills.

Repo, Commit, Branch
*Repo: The central storage for a project that contains all the files, folders, and the full history of changes.
*Commit: A snapshot of your code changes at a certain point of time.
*Branch: A parrallel and independent line of development within a repository.

Staging Area
*A crucial space in Git where a person prepares and finetunes the changes before committing them to the repository.

Push vs. Pull
*Push: Uploads your local changes to the remote repository.
*Pull: Downloads the changes fromt he remote repository and then integrates them.

What .gitignore does?
*The .gitignore file specifies intentionally untracked files and directories that Git should ignore, preventing Git from tracking these changes to the files.

What README is for?
A README is the primary guide to introduce, explain, adn instruct the users on how to use, install, and/or contribute to the software, code, or datasets.
